Sequence Exhaustion Risk in PostgreSQL

Sequences approaching their maximum value — integer overflow causes INSERT failures.

What to look for

pct_used > 80% on bigint; any significant % on int (max = 2.1B).

The diagnostic query

SELECT
    schemaname,
    sequencename,
    data_type,
    last_value,
    min_value,
    max_value,
    increment_by,
    cycle,
    round(
        (last_value - min_value)::numeric
        / nullif(max_value - min_value, 0) * 100, 2
    )                                                                 AS pct_used,
    (max_value - last_value) / nullif(increment_by, 0)               AS values_remaining
FROM pg_sequences
WHERE NOT cycle
  AND last_value IS NOT NULL
ORDER BY pct_used DESC NULLS LAST
LIMIT 20;

How to fix it

ALTER SEQUENCE seq MAXVALUE new_max; or ALTER COLUMN type TO bigint.
