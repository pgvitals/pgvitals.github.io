Logical Replication Slot Lag in PostgreSQL

WAL accumulating for logical replication consumers.

What to look for

consumer_lag_size > 500 MB — risk of disk exhaustion.

The diagnostic query

Requires: logical replication slots configured

SELECT
    slot_name,
    plugin,
    database,
    active,
    active_pid,
    pg_size_pretty(
        pg_wal_lsn_diff(pg_current_wal_lsn(), confirmed_flush_lsn)
    )                                                                 AS consumer_lag_size,
    pg_wal_lsn_diff(
        pg_current_wal_lsn(), confirmed_flush_lsn
    )                                                                 AS consumer_lag_bytes
FROM pg_replication_slots
WHERE slot_type = 'logical'
ORDER BY consumer_lag_bytes DESC NULLS LAST;

How to fix it

Check consumer health; if consumer is gone: SELECT pg_drop_replication_slot('name').
