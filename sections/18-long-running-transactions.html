Long-Running Transactions in PostgreSQL

Open transactions blocking autovacuum and holding locks.

What to look for

xact_duration > 5 minutes

The diagnostic query

SELECT
    pid,
    usename,
    application_name,
    client_addr,
    state,
    wait_event_type,
    wait_event,
    now() - xact_start                                                AS xact_duration,
    now() - query_start                                               AS query_duration,
    left(query, 200)                                                   AS current_query
FROM pg_stat_activity
WHERE xact_start IS NOT NULL
  AND now() - xact_start > interval '5 minutes'
  AND pid <> pg_backend_pid()
ORDER BY xact_start ASC;

How to fix it

SELECT pg_terminate_backend(pid) after investigation; set idle_in_transaction_session_timeout.
