PostgreSQL Diagnostic Guides

40 common PostgreSQL problems — each with ready-to-run SQL and a fix.

Query Behavior

  • 01 Slow / Expensive QueriesTop queries ranked by total CPU time consumed since last stats reset.
  • 02 Temp File & work_mem PressureQueries spilling intermediate results to disk because work_mem is too small.
  • 03 Sequential Scan HotspotsTables hit mostly with full sequential scans — a missing index signal.
  • 04 N+1 PatternsFast queries called thousands of times — classic ORM N+1 symptom.
  • 05 JIT Compilation OverheadQueries where JIT compilation time exceeds its execution benefit.

Index Health

  • 06 Unused IndexesIndexes never used in a query scan since the last pg_stat_reset.
  • 07 Duplicate / Redundant IndexesMultiple indexes covering the exact same column set.
  • 08 Invalid IndexesIndexes left in invalid state — typically from a failed CREATE INDEX CONCURRENTLY.
  • 09 Missing Foreign Key IndexesFK columns without a supporting index — causes seq scans on joins and cascades.
  • 10 Index BloatIndexes with high fragmentation — free-space waste from deletes and updates.

Tables & Storage

  • 11 Table BloatTables with large amounts of dead / unreclaimable space.
  • 12 TOAST Table BloatOversized TOAST tables storing large column values (text, jsonb, bytea, arrays).
  • 13 Table & Index Size RankingLargest objects ranked by total, heap, index, and TOAST size.
  • 14 Table Access PatternsHeap vs index fetch ratio, write load, and dead tuple ratio per table.
  • 34 Partitioned Table HealthPartitioned tables, partition counts, and total sizes.

Vacuum & Stats

  • 15 Autovacuum Worker ActivityCurrently running vacuum workers and their per-table progress.
  • 16 Dead Tuple UrgencyTables accumulating dead tuples faster than vacuum clears them.
  • 17 Stale StatisticsTables with many row modifications since last ANALYZE — causes bad query plans.
  • 18 Long-Running TransactionsOpen transactions blocking autovacuum and holding locks.

Connections & Locks

  • 19 Connection SaturationCurrent connections vs max_connections — headroom remaining.
  • 20 Idle-in-TransactionSessions sitting idle inside an open transaction — silently hold locks and block autovacuum.
  • 21 Lock Wait TreeFull chain of who is blocking whom.
  • 22 Wait Events BreakdownWhat all active sessions are currently waiting on.

Replication

  • 23 Streaming Replication LagPer-standby write, flush, and replay lag.
  • 24 Logical Replication Slot LagWAL accumulating for logical replication consumers.
  • 25 Replication Slot WAL RetentionTotal WAL held on disk by ALL slots (streaming + logical).

Risk Signals

  • 26 XID Wraparound RiskDistance from transaction ID exhaustion (hard limit: ~2 billion XIDs).
  • 27 MultiXact ID Wraparound RiskDistance from MultiXact exhaustion — a separate counter used for row-level locks.
  • 28 Sequence Exhaustion RiskSequences approaching their maximum value — integer overflow causes INSERT failures.

Config & Health

  • 29 Key GUC Settings ReviewCritical configuration parameters and their source (default vs tuned).
  • 30 Buffer Cache Hit RatioHow often reads are served from shared_buffers vs disk, per table and globally.
  • 31 Checkpoint & WAL PressureWhether checkpoints are forced too frequently and whether backends are writing directly to disk.
  • 32 Database-Level SummaryPer-database throughput, cache hit ratio, deadlocks, and temp usage at a glance.
  • 33 WAL Generation RateWAL (Write-Ahead Log) generation volume and rate since stats reset.
  • 36 I/O Stats by Backend (pg_stat_io)I/O statistics broken down by backend type, target object, and context.

Critical Risk Signals

  • 35 Open Prepared TransactionsUncommitted prepared transactions (2PC/two-phase commit).

Inventory & Extensions

  • 37 Extension InventoryAll installed extensions with version and upgrade availability.
  • 38 Foreign Data Wrappers & Foreign TablesFDW servers, user mappings, and foreign tables.
  • 39 Function PerformanceExecution stats for user-defined functions and stored procedures.
  • 40 Schema Size BreakdownStorage consumed per schema — tables, indexes, and TOAST.
